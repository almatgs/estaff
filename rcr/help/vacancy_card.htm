Карточка вакансии

Карточка вакансии в E-Staff имеет следующую структуру:

Раздел Общие сведения:

Номер: табельный номер вакансии. При заведении вакансии вручную присваивается системой автоматически.

Наименование: Наименование вакансии. Если в общих настройках системы (Работа с вакансиями) включена опция В случае привязки вакансии к штатной должности принудительно устанавливать наименование вакансии равным наименованию штатной должности, данное поле будет совпадать с наименованием штатной должности, к которой относится вакансия.

Структурное подразделение (для кадровых служб компаний): Ссылка на объект Структурное подразделение, к которому относится вакансия. В случае, если в общих настройках системы (Штатная структура) включена опция Вести учет штатных должностей, данному полю автоматически будет присвоено подразделение, к которому относится выбранная штатная должность.

Клиент (для кадровых агентств): Ссылка на объект Организация, к которой относится вакансия.

Контактные лица: Список контактных лиц (ссылки на объект Физическое лицо), которым будут направляться на рассмотрение кандидаты по данной вакансии.

Одиночная: Опция позволяет вести подбор "одна вакансия -  один человек". При приеме кандидата на работу программа предложит закрыть эту вакансию.

На несколько человек: Опция позволяет вести подбор нескольких человек на однотипную вакансию. При включении данной опции в карточке вакансии появится раздел Детализация и кнопка Добавить требуемое количество человек на панели инструментов. На каждого требуемого человек будет заведена т.н. детализированная вакансия со своим набором параметров (дата открытия, основание появления, приоритет, срок поиска, рекрутер, финальный кандидат), хранящаяся в карточке основной вакансии. При приеме на работу кандидата по основной вакансии программа будет закрывать следующую по хронологии детализированную вакансию. Когда все детализированные вакансии будут закрыты, программа предложит закрыть основную вакансию.

Массовая (поток): Опция позволяет вести подбор неограниченного количества человек на поточную вакансию, которая, как правило, никогда не закрывается. Данная опция отключает в карточке вакансии детализацию и подсчет статистики по событиям. При необходимости можно отключить ведение подбора по заявкам такого типа, сняв флажок Разрешать массовые вакансии (поток) в разделе Администрирование->Общие настройки->Работа с вакансиями.

Дата открытия: Дата открытия вакансии. При создании новой вакансии автоматически устанавливается в значение текущей даты.

В случае детализированной вакансии (На несколько человек), поле Дата открытия будет автоматически заполняться датой открытия первой открытой детализированной вакансии. При необходимости можно отключить данную опцию, сняв флажок Считать датой открытия вакансии на несколько человек дату открытия первой открытой детализированной вакансии в разделе Администрирование->Общие настройки->Работа с вакансиями.

Основание появления: Основание появления вакансии - замена, ввод новой штатной единицы, и т.п. (значение из справочника). Наполнение этого справочника можно изменить в разделе Администрирование->Справочники->Вакансии-Основания появления вакансий.

Начало поиска: Дата начала поиска соискателей по вакансии. Поле доступно при включенном режиме Считать датой начала работы по вакансии дату начала поиска, указываемую в отдельном поле в разделе Администрирование->Общие настройки->Работа с вакансиями.

Плановый срок: Максимальный срок поиска по вакансии. Может устанавливаться в днях, рабочих днях или месяцах. При изменении пользователем данного поля автоматически пересчитывается значение поля плановой даты закрытия вакансии - Закрыть до. В случае задания планового срока в рабочих днях плановая дата закрытия будет также определена как ближайший рабочий день.

Значение максимального срока поиска, устанавливаемого для новой вакансии по умолчанию, можно указать в разделе Администрирование->Общие настройки->Работа с вакансиями.

Корректировка: суммарное количество дней, добавленных к плановому сроку по различным причинам (вместо приостановки). Добавление сроков корректировки с возможностью указания типовых причин (например, продлен срок рассмотрения руководителем, продлен срок проверки СБ, кандидат не вышел, и т.д.) и комментариев производится в разделе Корректировки срока. Управление опцией производится флажком Разрешать корректировки срока закрытия вакансии в разделе Администрирование->Общие настройки->Работа с вакансиями.

Закрыть до: Дата, до которой вакансия должны быть закрыта (или должен быть завершен процесс подбора). Значение даты рассчитывается с учетом корректировки срока закрытия.  При изменении данного поля пользователем значение поля Плановый срок автоматически пересчитывается.

Текущий статус: Текущий статус вакансии (значение из справочника). По умолчанию вновь созданная вакансия имеет статус "В работе". В случае интеграции E-Staff с корпоративным порталом или системой кадрового учета, в которой заявки на подбор может подавать руководитель, загруженная, но еще не принятая в работу вакансия может иметь статус "Новая заявка". Статус "Закрыта", как правило, присваивается системой автоматически при приеме кандидата, рассматриваемого по данной вакансии, на работу. При необходимости уведомления заказчика о переходе вакансии в тот или иной статус в разделе Администрирование->Справочники->Вакансии->Статусы вакансий для требуемого статуса нужно будет добавить действие Отправить E-Mail или Отправить SMS с указанием шаблона, по которому будет формироваться уведомление. Для уведомлений по E-Mail можно использовать стандартный шаблон письма «Уведомление руководителя об изменении статуса вакансии».

Этап подбора: Наименование этапа подбора (значение из справочника). Поле доступно для вакансий, находящихся в активном статусе. Значение поля автоматически вычисляется в зависимости от того, до какого этапа дошел один из кандидатов, рассматриваемых на данную вакансию.

Список этапов подбора:

•          Предварительный отбор. Изначальный статус.

•          Интервьюирование. Условие: есть хотя бы один кандидат с запланированным или состоявшимся интервью, и который продолжает рассматриваться в данный момент (с активным статусом по вакансии).

•          Интервьюирование у заказчика. Условие: есть хотя бы один кандидат с запланированным или состоявшимся интервью с заказчиком, и который продолжает рассматриваться в данный момент (с активным статусом по вакансии).

•          Финалист. Условие: есть хотя бы один кандидат с событием “Оффер”, и который продолжает рассматриваться в данный момент (с активным статусом по вакансии).

В случае, когда несколько этапов удовлетворяют указанным для них условиям, выбирается последний по списку статус. Настройка этапов подбора производится в разделе Администрирование->Справочники->Вакансии->Этапы подбора вакансии.

Категория подбора: Категория (сценарий) подбора по вакансии - массовый подбор, специалисты, высшее руководство (значение из справочника). В разделе Администрирование->Общие настройки->Работа с вакансиями возможно указать опции, задающие возможность массового подбора, для каждой из категорий.

Категория сложности: Категория сложности вакансии (значение из справочника).

Приоритет: Приоритет вакансии (значение из справочника).

Оклад: Размер зарплаты для вакантной должности, в случае, если он заранее известен.

Уровень зарплаты (от, до): Диапазон зарплаты, в пределах которого осуществляется поиск кандидатов.

Специальность: Классификатор объектов по специальностям (значение из справочника). По умолчанию программа использует двухуровневый справочник специальностей, поставляемый вместе с программой. Вы можете отредактировать стандартный справочник специальностей в разделе Администрирование->Справочники->Специальности) или импортировать его из подготовленного файла формата MS Excel.

Страна: страна (государство) будущего места работы (значение из справочника).

Регион: регион будущего места работы (значение из справочника). Используется в том случае, если компания ведет детальность в нескольких регионах.

Станция метро: станция метро, ближайшая к будущему месту работы (значение из справочника).

Финальные кандидаты: Список кандидатов, являющихся финальными по данной вакансии. Как, правило, поле заполняется программой автоматически. Статус кандидата, при наступлении которого кандидат становится финальным, задается в разделе Администрирование->Справочники->Типы событий. По умолчанию кандидат становится финальным при наступлении статуса "Оффер". Последний финальный кандидат будет отмечен в списке плюсом «+».

Доп. сведения: Произвольная дополнительная информация. Использовать это поле для хранения описания вакансии не рекомендуется, для этого существует стандартное приложение Внутреннее описание.

Отобрано кандидатов, Интервью состоялось и пр.: Статистика по ключевым событиям ("воронка подбора"), показанная в виде диаграмм. Позволяет визуально оценить этап работы над вакансией и эффективность подбора. Наличие тех или иных счетчиков событий для разных признаков завершения определяется флажком Вести счетчик события данного типа по вакансии в настройках статуса соответствующего события (раздел Администрирование->Справочники->Общие->Типы событий).

 

Левая панель:

Инициатор: Лицо, являющееся инициатором (заказчиком) вакансии. Используется обычно в случае наличия нескольких контактных лиц по вакансии для указания конкретного лица, подавшего данную заявку.

Заморожена: Количество дней, в которых вакансия находилась в статусе «Приостановлена». Программа автоматически заполняет данное поле при последующей смене статуса вакансии на «В работе». В случае, если в разделе Администрирование->Общие настройки->Работа с вакансиями указана опция показа срока поиска по вакансии в рабочих днях, количество дней on-hold будет показано как в календарных, так и в рабочих днях. Если было принято решение возобновить подбор по закрытой ранее вакансии, то после изменения статуса вакансии на «В работе» для обновления данных по количеству дней on-hold необходимо вызвать команду Сервис->Пересчитать дни on-hold по истории изменений.

В работе: Количество дней, которое вакансия находится в работе. После закрытия вакансии (процесс подбора завершен), название поля сменится на Факт. срок и будет содержать количество дней от даты открытия до даты закрытия вакансии (даты завершения подбора). Если вакансия еще находится в работе, поле содержит количество дней от даты открытия до текущей даты.  В обоих случаях количество дней on-hold не учитывается.

Рекрутер/Группа: Пользователь E-Staff, ответственный за вакансию, и соответствующая группа. При назначении вакансии другому пользователю последний получит уведомление об этом в подразделе События->Уведомления.

Ограниченный доступ: опция, включаемая этим флажком, позволяет сделать данную вакансию "эксклюзивной", т.е. видимой ограниченному числу пользователей: владельцу вакансии, пользователям, указанным в поле Разрешить доступ для пользователей в подразделе Доп.сведения и пользователям, имеющим повышенный уровень доступа (опция Полный доступ или Разрешен просмотр объектов с ограниченным доступом в настройках уровня доступа).

 

Раздел Доп. сведения:

Внешний ID: В случае, если вакансия была загружена из корпоративного портала или внешней системы кадрового учета, данное поле будет содержать идентификатор соответствующего объекта во внешней системе.

Конфиденциальная вакансия: Опция, включаемая этим флажком, позволяет запретить всем пользователям доступ к разделу, содержащему данные для публикации вакансии («Объявление в Интернет»).

Дополнительная публикация к другой вакансии: Флажок, обозначающий, что карточка вакансии используется только как альтернативная публикация к другой (исходной) вакансии, без ведения отдельного подбора. В этом случае помещенный в базу отклик становится кандидатом, отобранным на исходную вакансию, а не на ту, на которую он откликнулся.

Агентства, работающие над вакансией (в режиме "Кадровая служба компании"): Список агентств (ссылки на объект Организация), с которыми ведется работа по данной вакансии.

Источник подбора: Источник поступления финального кандидата (значение из справочника). Поле заполняется программой автоматически.

 

Раздел Требования к кандидату: Раздел содержит список полей, определяющих требования к потенциальным кандидатам на вакансию (образование, опыт работы, возраст, пол). Кроме того, в поле Набор тестов можно указать требуемый набор (выбирается из настраиваемого справочника), содержащий список тестов, которые должны будут выполнить кандидаты на этапе тестирования. Справочник наборов тестов заполняется пользователем в разделе Администрирование->Справочники->Интеграция->Наборы тестов.

Раздел Детализация: Раздел содержит список детализированных вакансий (при включенной опции На несколько человек).

Раздел История изменений: Раздел содержит список истории изменения статуса вакансии, если таковые имели место. История заполняется автоматически при смене статуса вакансии.

Раздел Объявление в Интернет: Раздел содержит данные о публикации вакансии на сайтах, в случае использования Интернет-модуля E-Staff. Подробнее можно прочитать в разделе Размещение вакансии в Интернет.

Раздел Корректировки срока: Раздел содержит список корректировок срока закрытия вакансии. Добавление сроков корректировки с возможностью указания типовых причин и комментариев производится с помощью кнопки Добавить.

Раздел Кандидаты в работе: Раздел содержит список кандидатов, отобранных на данную вакансию. Над списком автоматически формируются статистические показатели, отражающие текущий прогресс процесса подбора по этой позиции. В левой части выводятся общие показатели – общее количество кандидатов, отобранных на данную вакансию, сколько из них сейчас рассматривается, сколько было отклонено или включено в резерв. Правее выводятся показатели, отражающие количество кандидатов, находящихся на каждом из этапов (в каждом из статусов). Для получения детализации по любому из показателей, достаточно щелкнуть по нему мышью.

Раздел События по кандидатам: Раздел содержит список событий, относящихся к кандидатам, отобранным (рассматриваемым) на данную вакансию.

Раздел Приложения: Раздел содержит произвольные приложения к карточке вакансии, в том числе стандартное приложение "Внутреннее описание".
