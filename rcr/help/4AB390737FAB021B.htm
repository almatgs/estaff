Действия над кандидатами

В левой части карточки кандидата под заголовком Действия содержится набор кнопок, позволяющий отражать типовые операции, проводимые c кандидатом в процессе подбора. В сетевой версии программы возможно ограничить доступность отдельных операций для пользователей пределами их групп, указав их в поле Актуально только для группы в разделе Доп. сведения карточки настроек события (Администрирование->Справочники->Общие->Типы событий).

Выполнение каждой из операций вызывает занесение соответствующей записи (события) в личное дело кандидата и, как следствие, изменяет статус кандидата. Для событий, имеющих несколько вариантов завершения, будет доступна кнопка Результат. Т.о. после того, как назначенное событие фактически состоится, с помощью этой кнопки можно вызвать диалоговое окно для отметки результата, и при необходимости ввести комментарий. При выборе результата, подразумевающего необходимость принятия решения рекрутером, автоматически включается возможность установить напоминание через заданный промежуток времени либо на предопределенную дату и время. Режимы работы системы напоминаний регулируются в разделе Персональные настройки->Общие->Напоминания.

Список возможных событий и их поведение регулируется в разделе Администрирование->Справочники->Общие->Типы событий. Ниже приводится описание стандартных событий, поставляемых вместе с системой.

Кнопка Пометка позволяет добавить произвольный комментарий в личное дело кандидата. При наборе комментария для ввода текста с новой строки можно использовать сочетание клавиш SHIFT+Enter. Клавиша Enter или кнопка ОК завершает ввод комментария. Добавление пометки не меняет статус кандидата.

Кнопка Первый звонок используется для регистрации факта совершения звонка кандидату с отметкой его результата (например, запланировать интервью, отклонить, перезвонить позже и т.д.).

Кнопка Начальный email используется, когда необходимо договориться с кандидатом о телефонном звонке или обговорить время интервью.  Для этого на выбор вам будет предложено несколько шаблонов писем для случая, когда кандидат сам откликнулся на вакансию (активный кандидат), и когда кандидат был найден самостоятельно (пассивный кандидат). См. также: Отправка единичных писем.

Кнопка Тел. интервью позволяет запланировать телефонное интервью с кандидатом. Пользователи, использующие программный телефон или звонки через Skype, могут позвонить кандидату из его карточки путем нажатия на значок с изображением телефона и выбором команды Позвонить. После того, как телефонное интервью фактически состоится, необходимо будет нажать кнопку Результат, отметить флажок, отражающий итог проведенного интервью, и при необходимости ввести комментарий. Если не хотите вводить комментарий, то можете это опустить, нажав кнопку OK или на клавиатуре Enter. При наборе комментария для ввода текста с новой строки можно использовать сочетание клавиш SHIFT+Enter.

Кнопка Интервью позволяет запланировать очное интервью с кандидатом. После того, как интервью фактически состоится, необходимо будет нажать кнопку Результат, отметить флажок, отражающий итог проведенного интервью, и при необходимости ввести комментарий. См. также: Планирование нового интервью. По умолчанию данное событие является определяющим для перевода вакансии, на которую рассматривается кандидат, на этап подбора Интервьюирование.

Кнопка Резюме у заказчика создает письмо для заказчика (или для всех контактных лиц по вакансии), содержащее резюме кандидата. После отправки письма статус кандидата изменится на Резюме у заказчика. После получения обратной связи от заказчика необходимо будет нажать кнопку Результат, отметить флажок, отражающий итог рассмотрения резюме, и при необходимости ввести комментарий. Процесс получения обратной связи от заказчика может быть автоматизирован при наличии интеграции E-Staff с корпоративным порталом Вашей компании. В этом случае после принятия заказчиком решения по кандидату пользователь получит уведомление об этом в подразделе События->Уведомления.

Кнопка Интервью с заказчиком позволяет запланировать очное интервью кандидата с заказчиком. После выбора даты и времени планируемого интервью программа автоматически предложит отправить кандидату письмо по шаблону - уведомление о назначенном интервью с заказчиком со схемой проезда. Откорректировать шаблон и прикрепить файл схемы проезда можно в разделе Администрирование->Справочники->Общие->Шаблоны писем. После того, как интервью фактически состоится, необходимо будет нажать кнопку Результат, отметить флажок, отражающий итог проведенного интервью, и при необходимости ввести комментарий. По умолчанию данное событие является определяющим для перевода вакансии, на которую рассматривается кандидат, на этап подбора Интервьюирование у заказчика. Дополнительно возможно использование интеграции календаря E-Staff с внешним календарем, используемым в компании, например, календарем Microsoft Outlook, Lotus Notes или Google (Персональные настройки->Календарь). Это позволит заказчику использовать механизм планирования даты и времени интервью в привычном для него календаре.

Кнопка Оффер используется для регистрации того факта что кандидату было сделано предложение. Статус кандидата изменится на Сделан оффер. После получения обратной связи от кандидата необходимо будет нажать кнопку Результат, отметить флажок, отражающий итог рассмотрения предложения, и при необходимости ввести комментарий. По умолчанию данное событие является определяющим для перевода вакансии, на которую рассматривается кандидат, на этап подбора Финалист.

Кнопка Проверка СБ регистрирует факт отправки кандидата на проверку в службу безопасности. Статус кандидата изменится на Проверка СБ. После получения обратной связи от службы безопасности необходимо будет нажать кнопку Результат, отметить флажок, отражающий итог проведенной проверки, и при необходимости ввести комментарий. Процесс направления кандидата на проверку и получения обратной связи может быть автоматизирован в случае интеграции E-Staff с корпоративным порталом Вашей компании.

Кнопка Оформление регистрирует факт запуска процедуры оформления кандидата на работу. Программа при этом предложит указать планируемую дату выхода кандидата на работу и продолжительность испытательного срока (при необходимости). Продолжительность испытательного срока по умолчанию, а также возможность указывать его индивидуально для каждого кандидата, регулируются в разделе Администрирование->Общие настройки->Работа с вакансиями.

В случае использования интеграции с внешней системой для загрузки данных о сотрудниках компании в E-Staff, статус Испытательный срок не пройден может быть присвоен кандидату автоматически в случае увольнения сотрудника во время прохождения им испытательного срока. См. Автоматическое формирование связки кандидата и сотрудника.

Статус кандидата изменится на Проверка СБ. После получения обратной связи от службы безопасности необходимо будет нажать кнопку Результат, отметить флажок, отражающий итог проведенной проверки, и при необходимости ввести комментарий.

Кнопка Принят на работу используется для выбора окончательного кандидата на данную вакансию. После выполнения данного действия программа предложит:

  • закрыть вакансию, на которую рассматривался кандидат, переведя ее в статус Закрыта или Закрыта заказчиком (на выбор)
  • перевести остальных кандидатов, по которым была начата работа по этой вакансии, в статус Не прошел финальный отбор
  • автоматически отправить по email непрошедшим кандидатам уведомление об отказе.

Управление запросом подтверждения пользователем на выполнение данных операций производится флажками Спрашивать подтверждение на смену статуса не прошедшим на вакансию кандидатам и Спрашивать подтверждение на отправку уведомления не прошедшим на вакансию кандидатам в карточке настроек статусов Закрыта и Закрыта заказчиком в разделе Администрирование->Справочники->Вакансии-Статусы вакансий.

После принятия кандидата на работу, за 5 рабочих дней до окончания испытательного срока, для рекрутера будет создана задача с напоминанием о необходимости запуска процедуры оценки прохождения испытательного срока. Для запуска процедуры оценки, в личном деле кандидата под событием Оценка прохождения испытательного срока (событие будет создано программой автоматически) необходимо будет нажать кнопку Результат и отметить флажок Начато. На электронный адрес руководителя посредством автоматической отправки почты будет отправлено уведомление о дате окончания испытательного срока сотрудника с просьбой выполнить оценку. После получения от руководителя обратной связи необходимо будет снова нажать кнопку Результат, отметить флажок Завершено, и при необходимости ввести комментарий. Интервал до окончания испытательного срока задается в доп.сведениях события Оценка прохождения испытательного срока в разделе Администрирование->Справочники->Общие->Типы событий.

Кнопки Отклонен рекрутером, Отклонен заказчиком и Самоотказ используются для регистрации факта отклонения кандидата на любом из этапов подбора, что прекращает рассмотрение кандидата по вакансии, на которую он рассматривался. При отклонении рекрутером или заказчиком программа предложит отправить кандидату уведомление об отказе. В случае самоотказа программа предложит указать причину и внести дополнительные комментарии. Набор типовых причин самоотказа регулируется в разделе Администрирование->Справочники->Кандидаты->Причины самоотказа кандидатов.

  

Кнопка Кадровый резерв используется для отправки кандидата в резерв, что прекращает его рассмотрение по вакансии, на которую он рассматривался.

 

Дополнительные кнопки действий:

Кнопка Запись на групповое интервью используется для регистрации кандидата на одно из запланированных групповых интервью по определенной вакансии. По умолчанию программа выберет ближайшее групповое интервью по требуемой вакансии, и занесет кандидата в список участников.

Кнопка Тестирование позволяет назначить кандидату тестирование, выбрав требуемый тест кнопкой Добавить тесты. Кроме того, возможно автоматическое назначение кандидату заданного набора тестов, если такой набор был указан в карточке вакансии в разделе Требования к кандидату. Создать наборы тестов можно в разделе Администрирование->Справочники->Интеграция->Наборы тестов. Как правило, тестирование проводится автоматически, если E-Staff интегрирован с системой тестирования, например WebTutor или Talent Q. Также можно вводить результаты тестирования вручную. Для каждой вакансии можно задать необходимый набор тестов. По окончанию тестирования пользователь получит уведомление об этом в подразделе События->Уведомления.

Кнопка Онлайн-интервью позволяет запланировать сеанс удаленного интервью с кандидатом, например, по Skype. Щелкнув мышью по значку Skype, вы сможете сразу перейти от просмотра карточки кандидата к звонку через Skype. После того, как онлайн-интервью фактически состоится, необходимо будет отметить флажок Онлайн-интервью состоялось - программа предложит ввести комментарий к интервью. Если не хотите вводить комментарий, то можете это опустить, нажав кнопку OK или на клавиатуре Enter.

Кнопка Утверждение руководителями начинает процедуру утверждения кандидата руководителями. Полученный от каждого из участников ответ (утвержден кандидат или нет) отмечается соответствующим флажком в подразделе Утверждение участниками карточки данного события. При положительном ответе всех участников кандидат автоматически перейдет в статус Утверждение руководителями - пройдено, а при наличии хотя бы одного отрицательного ответа - Утверждение руководителями - не пройдено.

Кнопка Оценка руководителями начинает процедуру оценки кандидата несколькими оценивающими руководителями с использованием определенного опросного листа. Листы для проведения оценки со списком вопросов создаются в разделе Администрирование->Справочники->Прочие->Оценочные листы. Полученные от каждого из участников результаты оценки кандидата заполняются в подразделе Ответы участников карточки данного события. В подразделе Общие сведения будут показаны усредненные по всем участникам значения результатов опроса. Как правило, направление оценочных листов руководителям и сбор результатов осуществляется автоматически, если E-Staff интегрирован с системой WebTutor.

Кнопка 2-е интервью с заказчиком позволяет запланировать повторное очное интервью кандидата с заказчиком. После того, как интервью фактически состоится, необходимо будет нажать кнопку Результат, отметить флажок, отражающий итог проведенного интервью, и при необходимости ввести комментарий.

Кнопка Принят по временному договору используется для выбора окончательного кандидата на данную вакансию, но на временной основе. После выполнения данного действия программа предложит закрыть вакансию, на которую рассматривался кандидат, перевести остальных кандидатов, по которым была начата работа по этой вакансии, в статус Не прошел финальный отбор и автоматически отправить этим непрошедшим кандидатам письмо по шаблону - уведомление об отказе.

Кнопка Уволен регистрирует факт увольнения ранее приятного на работу кандидата. В случае использования интеграции с внешней системой для загрузки данных о сотрудниках компании в E-Staff, статус Уволен может быть присвоен кандидату автоматически в случае увольнения сотрудника после завершения испытательного срока. См. Автоматическое формирование связки кандидата и сотрудника.

Кнопка Черный список заносит кандидата в черный список.

В сетевой версии программы, по умолчанию, при назначении события в карточке кандидата другого пользователя, оно будет отнесено к пользователю, указанному в карточке кандидата в поле Рекрутер. Если необходимо изменить поведение программы таким образом, чтобы назначаемое событие было отнесено к текущему пользователю, в разделе Администрирование->Общие настройки->Работа с кандидатами снимите флажок При назначении события кандидату другого пользователя, относить событие к пользователю, указанному в карточке кандидата.
