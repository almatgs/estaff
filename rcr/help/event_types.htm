Типы событий

Справочник Типы событий содержит список всех возможных типов событий, используемых в системе.

Большинство типов событий обозначает соответствующий статус объекта, к которому оно относится (например, Интервью состоялось, Резюме у руководителя, Испытательный срок). Также тип события может не обозначать никакого статуса, (например Пометка, Встреча).

Общие сведения:

Поле Базовый объект обозначает тип объекта, к которому относится событие. Для большинства событий в E-Staff Рекрутер базовым объектом является кандидат.

Флажок Определяет статус объекта обозначает, что последнее по хронологии событие данного типа обозначает статус объекта.

Флажок Запись в календаре применительно к кандидатам обозначает, что при заведение нового события предварительно откроется окно календаря для выбора даты и времени события.

Переключатель Единовременное событие/Временной интервал обозначает, является ли событие (статус) единовременным (например, Отклик на вакансию) или длящимся во времени (например, Интервью, Оффер).

Флажок Использовать комнаты обозначает, что для события данного типа можно выбрать комнату из списка комнат, с тем чтобы учитывать их занятость.

Флажок Привязывать к контрагенту обозначает, что в карточке соответствующего события появится поле Компания, содержащее ссылку на объект Организация.

Флажок Привязывать к вакансии обозначает, что в карточке соответствующего события появится поле Вакансия, содержащее ссылку на объект Вакансия.

Флажок Вести список участников обозначает, что в карточке соответствующего события будет храниться отдельный список участников по данному событию.

Флажок Утверждение несколькими участниками обозначает, что для включения признака завершения "состоялось" необходимо положительное решения каждого из участников события.

Флажок Заполнение участниками оценочной формы обозначает, что каждый их участников должен заполнить указанный опросный лист.

 

Область Признаки завершения содержит список возможных признаков завершения для данного события. Набор этих признаков зависит от сущности события.

Событие в общем случае может планироваться на будущее (признак «Запланировано»), затем оно наступает (признак «Состоялось»). Если оно длится определенное время, то оно может пройти успешно (признак «Пройдено») или неуспешно (признак «Не пройдено»).
Дополнительно событие может быть отменено (признак «Отменено»).

Кроме стандартных, при необходимости можно добавлять пользовательские признаки завершения (Добавить признак завершения->Пользовательский), но перед этим убедитесь, что ни один из стандартных признаков Вам действительно не подходит.

 

Доп. сведения:

Флажок Показывать отдельной кнопкой в левой части карточки кандидата обозначает, что событие входит в основную цепочку рассмотрения кандидата, и будет отображаться в левой части карточки кандидата отдельной кнопкой.

Поле Заголовок кнопки становится доступным для изменения при включенной опции Показывать отдельной кнопкой в левой части карточки кандидата.

Флажок Создать отдельный раздел программы для событий данного типа обозначает, что в разделе События добавится соответствующий подраздел для выноса списка событий данного типа.

Поле Актуально только для группы позволяет ограничить в карточке кандидата доступность события данного типа для пользователей пределами их групп.

Флажок Выгружать во внешний календарь обозначает, что при назначении события его копия будет выгружаться во внешний календарь того пользователя, который сохранил карточку события. Функция будет активна при включенной опции Запись в календаре в разделе Общие сведения, выбранном внешнем календаре в разделе Персональные настройки->Календарь и установленном модуле в разделе Персональные настройки->Модули к внешним программам (при использовании MS Outlook или Lotus Notes).

Флажок Предварительно выбирать время, используя встроенный календарь обозначает, что для выбора времени планируемого события будет использован внутренний календарь программы.

Флажок Открывать событие во внешнем календаре обозначает, что при заведении события соответствующего типа будет создано и открыто событие во внешнем календаре пользователя. Если этот флажок снят, но включена опция Вести список участников в разделе Общие сведения, то программой будет произведена автоматическая отправка уведомлений участникам мероприятия об этом событии.

 

Статусы:

Поле Стандартное наименование определяет наименование соответствующего статуса объекта. По умолчанию наименование статуса объекта определяется как наименование события, плюс признак завершения.

Флажок Деактивирует объект обозначает, что объект с данным статусом переходит в неактивное состояние (обозначается серой картинкой). Примеры таких статусов - Кадровый резерв, Отклонен рекрутером, Принят на работу.

Флажок Статус подразумевает необходимость принятия решения пользователем для события по кандидату обозначает, что ответственным за то, что кандидат до сих пор находится в данном статусе, является рекрутер. Например, статусы Отклик на вакансию, Интервью состоялось, Резюме одобрено заказчиком обозначают, что рекрутер должен либо отклонить кандидата, либо предпринять какие-либо активные действия по дальнейшей работе с кандидатом и переводе его в следующий статус. Кандидаты с данными статусами будут отображаться в разделе Кандидаты->Принять решение (при условии нахождения в данном статусе не более 90 дней).

Флажок Создать отдельный раздел программы для объектов сданным статусом применительно к кандидатам обозначает, что в основном разделе Кандидаты по статусу появится соответствующий раздел.

Флажок Запрашивать подтверждение пользователя означает, что при переводе в данный статус будет появляться диалоговое окно с запросом на подтверждение.

Флажок Показывать диалоговое окно для ввода комментария означает, что при переводе в данный статус будет появляться диалоговое окно для ввода комментария.

Флажок При переводе в данный статус открывать карточку события для ввода дополнительной информации применительно к кандидатам обозначает, что при переходе события в данный статус откроется карточка события, в которую можно будет внести дополнительную информацию.

Флажок Наличие комментария обязательно обозначает, что система не разрешит добавить событие без заполнения поля Комментарий комментарием минимально указанной длины (поле Минимальная длина комментария).

Поле Начальный шаблон комментария позволяет задать некоторый начальный шаблон, чтобы при появлении диалогового окна для ввода комментария оно формировалось на основе этого шаблона.

Поле Предельный срок нахождения определяет максимальный нормативный срок, в котором объект может находится в данном статусе. Если объектом является кандидат, то истечении предельного срока он будет отображаться в разделе Кандидаты->Статус просрочен (при условии нахождения в данном статусе не более 90 дней).

Флажок Требует привязки кандидата к вакансии обозначает, что система не позволит перевести кандидата в данный статус, пока кандидат не будет отобран на какую-либо активную вакансию или пока вакансия не будет предварительно выделена в списке Рассматривается на вакансии (в случае рассмотрения кандидата на несколько активных вакансий).

Флажок Определяет кандидата в качестве финального по вакансии обозначает, что, при наступлении указанного статуса кандидат помечается в качестве финального кандидата по той вакансии, на которую он рассматривается.

Флажок Изменяет статус вакансии обозначает, что, при наступлении указанного статуса меняется статус вакансии, на которую рассматривается кандидат.

Флажок При наступлении автоматически переводит объект в следующий статус обозначает, что при наступлении данного статуса объект будет автоматически переходить в следующий указанный статус. Например, при наступлении статуса Резюме одобрено руководителем кандидат может переводиться в статус Оценка руководителями.

 

Подраздел Действия:

Тип действия - ссылка служит для указания дополнительного действия (Отправить E-mail или Выполнить скрипт), которое необходимо выполнить перед/после перехода события в данный статус.

Отправить E-mail обозначает, что при переходе события в данный статус объекту будет отправлено сообщение по указанному шаблону. Кроме того, есть возможность задания нескольких альтернативных шаблонов для письма с помощью кнопки Выбор элемента справочника.

Флажок Отправлять участникам обозначает, что при переходе в данный статус участникам события будет автоматически сформировано сообщение по указанному шаблону для отправки. Опция становится доступной при установленном флажке Вести список участников в разделе Общие сведения.
