Soovitused süsteemi seadistamiseks ametistruktuuriga tööks suurtes (holding) ettevõtetes  

Suured (holding) ettevõtted omavad oma mastaapsuse tõttu sageli väga keerulist hierarhilist organisatsioonilist struktuuri. Sellisel juhul võib täielik üksuste kett olla väga pikk, näiteks:

Ettevõte OÜ ->  Tootmistoodete divisjon -> Arendusosakond -> Ostugrupp -> Transpordi sektor

Sellisel keti pikkusel ei näe me sellele määratud kaardi väljal või nimekirjades sihtstruktuurüksust (toodud näides – üksust „Transpordi sektor“), vaid ainult keti algust. On olemas kaks teineteist täiendavat lahendust antud probleemile: 

  • Üksuste lühendatud nimetuste kasutamine (vaikimisi). Väli Lühendatud nimetus täidetakse struktuurüksuse kaardil.

Meie näides oleks tulemus umbes selline:

Ettevõte OÜ -> TT Divisjon -> Arendusosakond -> OstuGr-> TranspSektor

Sellise funktsiooni haldamist teostatakse sektsioonis Administreerimine->Üldised seadistused ->Ametistruktuur valikuga Täieliku üksuste keti kuvamisel kasutada üksuse lühendatud nimetust, juhul kui see on määratud. 

  • Üksuste hierarhilise keti kuvamise režiimi ümberlülitamine vastupidiseks.

Meie näide jaoks oleks tulemus selliseks:

Transpordi sektor <- Ostugrupp <- Arendusosakond <- Tootmistoodete divisjon <- Ettevõte OÜ

Ning mõlema variandi kasutamisel:

TranspSektor<- OstuGr <- Arendusosakond <- TT Divisjon <- Ettevõte OÜ

Sellise funktsiooni haldamist teostatakse sektsioonis Administreerimine->Servis->Objektid->Struktuurüksus valikuga Nimetuse kuvamise kord hierarhias.
